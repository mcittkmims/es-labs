\subsection{System Observations}

The implementation uses a 2000 ms acquisition period because the DHT22 should not be polled aggressively. This period is acceptable for room-temperature control, where thermal changes are slow compared with MCU execution time. The relay output is fail-safe: invalid sensor reads force the actuator OFF. The potentiometer provides fast analog setpoint adjustment, while the keypad gives repeatable manual values and hysteresis-band control.

The Wokwi project is retained as a wiring reference, but the missing battery in the simulator does not affect firmware behavior because the MCU only controls the relay coil input. The switched load branch is electrically separate from the logic branch and must be validated on the physical setup.

\subsection{Verification Summary}

\begin{itemize}
    \item \textbf{Compilation:} `lab5\_1` builds successfully for Arduino Mega 2560.
    \item \textbf{Acquisition:} DHT22 temperature and humidity are sampled in a dedicated task.
    \item \textbf{Setpoint:} A0 potentiometer input is mapped to 15--35 degrees Celsius, with keypad override.
    \item \textbf{Control:} Hysteresis logic preserves relay state inside the dead band.
    \item \textbf{Actuation:} Relay command is applied from a separate task.
    \item \textbf{Display:} LCD and Serial Plotter output expose setpoint, value, thresholds, output, and validity.
    \item \textbf{Validation evidence:} Hardware photos and Serial Plotter screenshots must be added after running the real Arduino setup.
\end{itemize}
