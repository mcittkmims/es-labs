\section{Design}

\subsection{System Components}

The dual actuator control system consists of the following components, each with a clearly defined role in the architecture:

\begin{enumerate}
    \item \textbf{Arduino Mega 2560} --- central microcontroller running FreeRTOS, hosting all tasks and coordinating communication between modules.
    \item \textbf{Relay Module} --- binary actuator driven by digital pin 7, switches an external load (lamp) ON/OFF.
    \item \textbf{PWM Output (Pin 9)} --- analog actuator interface generating a variable duty cycle signal for motor/LED control.
    \item \textbf{Green LED (Pin 12)} --- status indicator mirroring the relay state.
    \item \textbf{Yellow LED (Pin 11)} --- status indicator for analog actuator activity.
    \item \textbf{Blue LED (Pin 9)} --- load simulation for the analog actuator (PWM-driven brightness).
    \item \textbf{LCD 1602 I2C} --- 16x2 character display showing both actuator states, debounce counters, and setpoints.
    \item \textbf{Serial Terminal (UART0)} --- command input interface and diagnostic report output.
    \item \textbf{BinaryConditioner} --- software module implementing counter-based debouncing for the relay command.
    \item \textbf{SignalConditioner} --- software module providing saturation, median filtering, and EWMA for the analog command.
    \item \textbf{FreeRTOS Scheduler} --- manages four concurrent tasks with priority-based preemptive scheduling.
\end{enumerate}

\subsection{Layered Software Architecture}

The software follows a layered architecture that separates concerns across five abstraction levels, as shown in \citeimg{fig:layered_architecture}. The Application Layer (APP) contains the FreeRTOS tasks that orchestrate the system behavior. The Service Layer (SRV) provides signal conditioning algorithms (BinaryConditioner, SignalConditioner) and command parsing logic. The ECU Abstraction Layer (ECAL) handles STDIO-to-UART redirection and I2C LCD communication. The Microcontroller Abstraction Layer (MCAL) wraps low-level GPIO, PWM, and I2C peripheral access through the Relay, PwmActuator, Led, and LcdDisplay drivers. The Hardware Layer (HW) represents the physical ATmega2560 and its connected peripherals.

\subsection{Hardware-Software Interface Architecture}

For each hardware interface, the layered architecture provides a clear mapping from physical connection to software API:

\begin{itemize}
    \item \textbf{Relay interface:} GPIO pin 7 $\rightarrow$ MCAL \texttt{Relay} class (\texttt{turnOn()}, \texttt{turnOff()}, \texttt{isOn()}) $\rightarrow$ SRV \texttt{BinaryConditioner} $\rightarrow$ APP Binary Task.
    \item \textbf{PWM interface:} PWM pin 9 $\rightarrow$ MCAL \texttt{PwmActuator} class (\texttt{setDutyCycle()}, \texttt{getDutyCycle()}) $\rightarrow$ SRV \texttt{SignalConditioner} with ramping $\rightarrow$ APP Analog Task.
    \item \textbf{LED interfaces:} GPIO pins 11, 12 $\rightarrow$ MCAL \texttt{Led} class (\texttt{turnOn()}, \texttt{turnOff()}) $\rightarrow$ APP actuator tasks.
    \item \textbf{LCD interface:} I2C bus (SDA=20, SCL=21) $\rightarrow$ ECAL \texttt{LcdDisplay} class (\texttt{showTwoLines()}, \texttt{printLine()}) $\rightarrow$ APP Display Task.
    \item \textbf{Serial interface:} UART0 (pins 0/1) $\rightarrow$ ECAL \texttt{StdioSerial} (\texttt{printf()}, \texttt{Serial.read()}) $\rightarrow$ APP Command Task.
\end{itemize}

\subsection{Modular Implementation}

\subsubsection{Relay Driver Module}

The \texttt{Relay} library provides a hardware abstraction for controlling a relay module via a digital GPIO pin. It supports both active-HIGH and active-LOW relay modules through a constructor parameter. The interface exposes four methods: \texttt{init()} configures the pin as OUTPUT and deactivates the relay, \texttt{turnOn()} and \texttt{turnOff()} set the relay state with automatic logic inversion for active-LOW modules, and \texttt{setState(bool)} provides a convenient boolean-controlled interface. The \texttt{isOn()} getter returns the current relay state for diagnostic reporting.

\begin{lstlisting}[language=C++, caption={Relay driver interface (Relay.h)}]
class Relay {
public:
    Relay(uint8_t pin, bool activeHigh = true);
    void init();
    void turnOn();
    void turnOff();
    void setState(bool on);
    bool isOn() const;
};
\end{lstlisting}

\subsubsection{PWM Actuator Driver Module}

The \texttt{PwmActuator} library abstracts PWM output control for analog actuators. It accepts a duty cycle as a percentage (0.0--100.0\%) and converts it to the 0--255 range used by Arduino's \texttt{analogWrite()} function. Input values are automatically clamped to the valid range. The \texttt{getDutyCycle()} and \texttt{getRawPwm()} getters provide diagnostic access to the current output state.

\begin{lstlisting}[language=C++, caption={PWM actuator driver interface (PwmActuator.h)}]
class PwmActuator {
public:
    PwmActuator(uint8_t pin);
    void init();
    void setDutyCycle(float percent);
    float getDutyCycle() const;
    uint8_t getRawPwm() const;
};
\end{lstlisting}

\subsubsection{Binary Conditioner Module}

The \texttt{BinaryConditioner} library implements counter-based debouncing as described in Section 4.1.6 of the lab manual. A counter increments when the raw input is HIGH and decrements when LOW, bounded by [0, MAX]. The output transitions to ON only when the counter reaches MAX, and to OFF only when it reaches zero. This prevents false triggering from transient noise or accidental rapid command changes. The module exposes diagnostic getters for the counter value and transitioning status.

\begin{lstlisting}[language=C++, caption={Binary conditioner interface (BinaryConditioner.h)}]
class BinaryConditioner {
public:
    BinaryConditioner(uint8_t maxCount = 5);
    void init();
    bool process(bool rawInput);
    bool getState() const;
    uint8_t getCounter() const;
    bool isTransitioning() const;
};
\end{lstlisting}

\subsubsection{Reused Modules}

The following library modules from previous labs are reused without modification:

\begin{itemize}
    \item \textbf{SignalConditioner} --- three-stage pipeline (saturation $\rightarrow$ median $\rightarrow$ EWMA) for conditioning the analog actuator setpoint.
    \item \textbf{LcdDisplay} --- I2C LCD driver with \texttt{showTwoLines()} for displaying both actuator states.
    \item \textbf{Led} --- GPIO LED driver for status indicators.
    \item \textbf{StdioSerial} --- STDIO redirection over UART for \texttt{printf()} output.
\end{itemize}
